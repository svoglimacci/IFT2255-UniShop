Descriptions des cas d'utilisation

Cas d'utilisation : Inscription en tant qu'acheteur
But : L'utilisateur doit fournir les informations necessaires telles que son nom, prenom, pseudo unique, adresse courriel, numero de telephone et l'adresse d'expedition. Il doit aussi etre capable de le modifier en tout temps.
Préconditions : Le compte de l'utilisateur n'existe pas.
Acteurs : Acheteur
Scénario principal : L'utilisateur veut se créer un compte
1. Étape 1: Menu d'inscription
 1.1. Étape 1.1 : La plateforme demande à l'utilisateur d'entrer les informations nécessaires.
2. Étape 2: Vérification la validité des informations
 2.1. Étape 2.1 : Pseudo unique
 2.2. Étape 2.2 : Adresse courriel valide
 2.3. Étape 2.3 : Numero de telephone valide
 2.4. Étape 2.4 : Adresse d'expédition valide
3. Étape 3: Confirmation de l'inscription de l'utilisateur
Scénarios alternatifs :
2a.1. Les informations saisies sont invalides
2a.2. La plateforme demande de corriger les informations invalides
3a.1. Si le compte existe déjà, la plateforme redirige l'utilisateur vers la page de connexion.
Postconditions : La plateforme crée un nouvel Utilisateur avec les informations données et la sauvegarde à la base de donnée.

 2.1. Étape 2.1: L'utilisateur saisie les informations requises
3. Étape 3

1b.1. autre remplacement de la même étape
1b.2. suit 1b.1 indépendamment du scénario principal
Postconditions : L'inventaire du vendeur du/des produit(s) achetés est mise à jour. Le vendeur est informé de l'achat. Chaque commande possède un identifiant unique.
